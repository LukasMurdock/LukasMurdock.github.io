- [Functionality is an asset. Code is a liability.](https://wiki.c2.com/?SoftwareAsLiability)
- [There is no silver bullet.](https://worrydream.com/refs/Brooks_1986_-_No_Silver_Bullet.pdf)
- [For each desired change, make the change easy (warning: this may be hard), then make the easy change](https://martinfowler.com/articles/preparatory-refactoring-example.html)
- [Reality has a surprising amount of detail. If you wish to not get stuck, seek to perceive what you have not yet perceived.](http://johnsalvatier.org/blog/2017/reality-has-a-surprising-amount-of-detail)
- [Iteration velocity solves all known problems in software engineering (including quality)](https://x.com/cramforce/status/1996623144422117486)
- [Complexity, very, very bad. Do not summon complexity demon.](https://grugbrain.dev/)
- [Choose boring technology.](https://boringtechnology.club/)
- [Scaling doesn’t scale.](https://www.marginalia.nu/log/65-scaling-doesnt-scale/)
- [Make your code usable before you try to make it reusable](https://caseymuratori.com/blog_0015) / [Duplication is far cheaper than the wrong abstraction.](https://sandimetz.com/blog/2016/1/20/the-wrong-abstraction) / [Avoid Hasty Abstractions](https://kentcdodds.com/blog/aha-programming)
- [many times, you can achieve the desired functionality simply with clever reconfigurations of the basic Unix tool set](http://widgetsandshit.com/teddziuba/2010/10/taco-bell-programming.html)
- What’s not pushed into the repository, does not exist.
- [The standard pace is for chumps. There is no speed limit.](https://sive.rs/kimo) / [Fast](https://patrickcollison.com/fast) / [It is possible to move fast and fast matters](https://brianguthrie.com/p/the-move-faster-manifesto/) / [When decisions are reversible, make them fast. When decisions are irreversible, slow them down.](https://fs.blog/reversible-irreversible-decisions/)
- [Yeet. Delete. Repeat.](https://jason.energy/yeet-delete-repeat/)
- Design for absence. Build systems that work without you—that work when you’re asleep, sick, or on vacation. Async communication. Self-service where possible. Documentation that answers questions before they’re asked. [Bus factor.](https://en.wikipedia.org/wiki/Bus_factor)
- [Don’t care what Google or Apple or whoever did](https://adrianroselli.com/2020/03/i-dont-care-what-google-or-apple-or-whomever-did.html)
